# 369-vault

**369（= IKEZAKI OS / LeadMap AI）の知識ベース（ナレッジ・ヴォルト）です。**

このリポジトリは、コード本体である `369` リポジトリとは別に、**思想・哲学・世界観・プロンプト・設計判断・用語**といった「消えやすいが最も価値のある知識」を Markdown で蓄積するための場所です。

## 目的

- **思想を残す**: なぜこのプロダクトを作るのか、どんな世界を目指すのかを言語化して保存する。
- **プロンプトを資産化する**: 実際に使って効果のあったプロンプトを1プロンプト1ファイルで蓄積し、再利用できるようにする。
- **知識を構造化する**: 設計判断・用語・意思決定の理由を、後から誰でも辿れる形で残す。

## 読み方（Obsidian 前提）

- このリポジトリは **[Obsidian](https://obsidian.md/)** で開くことを前提にしています。
- ノート同士は `[[ノート名]]` の **Obsidian リンク**で相互参照しています。
- まず **[[index]]** を開いてください。全フォルダへの入口になっています。
- すべて **日本語 Markdown**で書かれています。

## フォルダ構成

```
369-vault/
├── index.md          … トップの目次ノート（ここから辿る）
├── README.md         … このファイル（リポジトリの目的説明）
├── 思想/             … 369 の思想・哲学・世界観・ビジョン
├── プロンプト/       … 実際に使っているプロンプト集（1プロンプト1ファイル）
└── 知識/             … 設計判断・用語集・ナレッジ・意思決定の記録
```

## 更新ルール

- `369` 本体で**知識・思想・プロンプトを更新／追加したら、その内容をこの `369-vault` にも Markdown で反映**し、commit & push する（`369` の `CLAUDE.md` にも明記）。
- 新しいノートを足したら、必ず **[[index]]** からリンクを張って迷子ノートを作らない。
- 一時的な状態（未push・承認待ち等）はここに固定しない。**現在地は `369` リポジトリの `tasks/CURRENT_STATE.md` と git を正**とする。

> 注: このヴォルトは「コードの外にある頭脳」です。コードが how を語り、ここが why を語ります。
