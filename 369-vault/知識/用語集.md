# 用語集

> 目次に戻る → [[index]] ／ 関連 → [[アーキテクチャ概要]] ・ [[UsageEvent非課金台帳]] ・ [[セキュリティと権限]]

このプロダクト特有の言葉の意味。

## プロダクト

- **IKEZAKI OS** — 中小企業向けの統合AI経営OS（経営/営業/CRM/会計/財務/人事/在庫/会議/ナレッジ/AI社員）。
- **LeadMap AI** — 地図×AIの新規開拓OS。リード抽出→AI分析→個別営業メール→承認→送信→追客→商談化。
- **Self-Evolving Agentic Business OS** — 目指す姿。AIが業務を支援・代行・報告・改善する経営基盤。

## 記録・監査

- **UsageEvent** — 非課金の「利用量」記録。`billing=usage_only`。金額カラムを持たない。→ [[UsageEvent非課金台帳]]
- **AuditLog / writeAudit** — 重要操作の監査ログ。
- **DataAccessLog / writeDataAccess** — 機密データ参照（AI参照含む）の記録。
- **ApprovalRequest** — 危険操作の承認リクエスト。外部送信/契約/請求/支払/削除/エクスポート/人事は承認必須。

## 権限・安全

- **RBAC** — `packages/shared/rbac.ts`。10ロール×8アクション、ワイルドカード対応。
- **機密ラベル (labels)** — `packages/shared/labels.ts`。
- **maskText** — 外部LLM送信前などに使うマスク処理。
- **safeFetch** — 外部URLフェッチの唯一の口。SSRFガード/allowlist/timeout/サイズ上限。
- **requiresApproval** — 危険操作に承認を強制する仕組み。
- **EXTERNAL_SEND_ENABLED** — 外部送信の可否フラグ（既定 false＝安全）。

## AI

- **FakeLLM / DemoMapProvider** — APIキー無しでフル機能デモを可能にする擬似実装。
- **getLLMProvider / getEmbeddingProvider / getTranscriptionProvider** — env 未設定で自動 Fake。
- **OutreachDraft** — AI生成の営業文（必ず下書き。送信は承認後）。

## メール営業

- **ConsentRecord** — 同意管理。
- **SuppressionList** — 配信停止・抑止リスト。送信前に `isSuppressed` で確認。
- **OutreachSendLog** — 送信ログ。
- **detectUnsubscribeRequest** — 返信から配信停止希望を検知。

## 開発・状態

- **薄い縦切り** — テーブル/API/UI/デモデータ/権限/監査を1本通す最小実装単位。
- **docs-only フェーズ** — コードを触らず設計・監査・記録だけ行うフェーズ。
- **PROGRESS.md / CURRENT_STATE.md** — 履歴 / 現在地の1枚サマリー。→ [[状態管理修復プロンプトの型]]
- **BLOCKERS.md** — 環境制約と回避策（B-01 Prisma エンジン等）。→ [[既知の落とし穴とローカル検証]]
- **tenantId** — 全モデルが持つテナント識別子。クエリは必ず tenantId でスコープ。

## Phase 3 / Control Tower

- **AI Growth Opportunity Control Tower** — Phase 3（AI Growth Engine）最初の縦切り。社長向けに成長機会カード9枚を見せる read-only 中心の画面（`/growth/control-tower`）。→ [[ControlTowerV0設計と実装前Gate]]
- **redaction（財務伏せ字）** — 財務閲覧権限のない担当者に原価・粗利・未回収の実値や finance 件数を見せない不変条件。「書くとき」と「見せるとき」の二重防御で守る。→ [[push前敵対的レビューの型]]
- **stage3_e2e** — CI の第3段。一時 DB（pgvector）＋seed＋build＋Playwright 実走で全 e2e を回す回帰ゲート。封印 env（fake/log/false）もここで毎回確認。→ [[CIStage3E2E設計と実装]]
- **実装前 Gate** — 実装に入る前に「既存 schema・RBAC・seed だけで成立するか」を docs-only で判定し、成立しないなら STOP する関門。→ [[実装前Gateプロンプトの型]]
- **敵対的レビュー** — push 前の commit を複数視点の独立 AI レビューで総点検し、確定した穴は修正＋見張りテスト追加までをセットで行う型。→ [[push前敵対的レビューの型]]
- **P3-CT-0〜7** — Control Tower v0 の段組み（設計→read-only 画面→優先度→監査ログ→AI 下書き→承認導線→e2e→push/CI）。→ [[P3CT4完全クローズとP3CT5準備]]
