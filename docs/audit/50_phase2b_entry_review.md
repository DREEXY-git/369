## 4. 3領域の比較表

| 2-Aの安全境界の流用 | **ほぼそのまま流用可**（参照構造も doc33 §5 で設計済み） | 流用可＋**匿名化フラグ・許諾の追加設計が必要** | 流用だけでは不足（**高機密ラベル対応が前提**） |

## 5. PII・許諾・外部発信・ENSHiN OS リスク評価

- **Sales Playbook**: 顧客の声・口コミ・SNS・外部発信のいずれにも直接触れない。ENSHiN OS 安全観点は「プレイブック本文に顧客名・事例を書き込まない」入力ガイドだけで足りる見込み。
- **Case Study**: **ENSHiN OS の「顧客の声」「導入事例」「感謝の声」「testimonial」「紹介」「関係資産」領域と近接**。実名・写真・成果数値・Before/After を扱う可能性があり、**外部公開・SNS投稿案・ブランディング素材化・口コミ依頼**への転用が構造的に起こり得る。したがって Case Study を実装する場合は、**許諾管理（ConsentRecord 連携）・公開前の人間承認・広告表現チェック（No.1表記・効果保証・誇大表現は人間/法務判断へ）・外部発信ログ**の設計が**同時に必須**（doc49 §10 の停止条件がそのまま適用される）。それまでは**社内参照専用＋匿名化前提**に限定する。
- **Customer Pain**: 公開素材ではなく、高機密（CUSTOMER_CONFIDENTIAL 相当）前提。現状の Company Brain は NORMAL / INTERNAL のみ対応（高機密は保留中）のため、**高機密ラベル対応（writeDataAccess 済みの現在も、ラベル解禁自体が別の重い承認）が先**。
- **Enshin OS の詳細仕様は本リポジトリに未提供（doc07・証拠不足のまま）**。上記の近接評価は「領域の性質」に基づく整理であり、Enshin OS の個別機能を推測で断定していない。
- 共通: **口コミ投稿・SNS投稿・顧客の声公開・推薦コメント掲載・紹介依頼送信・社長/会社ブランディング投稿は、本レビューでも今後の 2-B でも AI の自己判断では一切行わない**（下書き・承認・許諾・ログまで）。

## 6. Phase 2-A から流用する安全境界

Phase 2-B の全候補に、実証済みの以下をそのまま適用する:

- 全モデルに tenantId スカラ（クエリは必ず tenantId スコープ）
- archivedAt によるソフトアーカイブのみ（delete/deleteMany なし）
- ラベルは **NORMAL / INTERNAL のみ**（高機密ラベルは個別の重い承認まで保留）
- **externalAiAllowed は false デフォルト・true にする UI を作らない**（外部LLM時は true＋maskText のみ注入＝構造的にゼロ）
- **AI mutation 禁止**（actions 層で人間専用化・rbac.ts は変更しない）
- writeAudit（変更系）＋ writeDataAccess / **ai_reference**（AI参照はレコードごと1件）
- 三段承認（設計 docs → schema 変更 → 実装、それぞれ個別人間承認）

## 8. 推奨する Phase 2-B-1 の最小スコープ（1つに絞る）

**推奨: Phase 2-B-1 = SalesPlaybookEntry の設計 docs-only（schema 設計案の作成のみ・実装なし）。**

理由（非エンジニア向け）:
1. **いちばん安全**: 顧客の個人情報から最も遠く、許諾・匿名化・高機密のどれも前提にしない。失敗してもお客様に影響しない。
2. **型がそのまま効く**: Phase 2-A の CompanyPolicy とほぼ同じ作り（タイトル・本文・カテゴリ・タグ・ラベル2択・アーカイブ）＋「どの商品・どの方針に紐づく売り方か」の参照を足すだけ。doc33 §5 に参照構造の素案が既にある。
3. **価値が早い**: ナレッジ検索の AI 回答（Company Brain 参照）に「売り方の型」が加わり、営業準備・新人教育にすぐ効く。
4. **ENSHiN OS を焦らない**: 顧客の声・事例（Case Study）は魅力的だが、許諾管理と公開前承認の部品が先。順番を守ることで「許諾なし公開」を構造的に起こさない。

Phase 2-B-1 の範囲（次の人間承認の対象）: SalesPlaybookEntry の**設計 docs のみ**（フィールド案・ラベル方針・参照構造・AI参照の扱い・三段承認の計画）。schema 変更・migration・実装・seed は含まない。
