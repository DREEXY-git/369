# Phase X-05-1 — GitHubに上げるたびに自動チェックが走るようになる

> 出典: `369` リポジトリ `docs/audit/64_phase_x05_1_ci_stage1.md`（判定 GO・commit-only）。
> 関連: [[PhaseX05CI否定系テスト設計]] / [[Phase2B完了]]

## 何が変わるか

- これまで品質チェック（テスト211本・型チェック・lint）は、作業のたびに**手動で実行する規律**に頼っていた。
- 今回、**GitHub にコードが上がるたびに同じチェックが自動で走る仕組み（CI）**を追加した。壊れた変更は main に入る前に**赤信号**で分かるようになる。
- 追加したのは設定ファイル**1つだけ**（`.github/workflows/ci.yml`）。アプリのコード・データベース・本番環境・秘密情報には一切触れていない。

## 中身（Stage 1 = 軽くて速いチェックだけ）

1. 依存パッケージを厳密に再現インストール
2. データベースクライアント生成
3. テスト211本
4. 型チェック
5. コードスタイル検査（lint）

設計（[[PhaseX05CI否定系テスト設計]]）どおり、重いもの（ビルド・ブラウザテスト・DB起動）は Stage 2/3 として**別承認**に残してある。

## まだやっていないこと

- **GitHub 上で実際に動くかの確認（実走確認）は push 後**。GitHub の「Actions」タブに緑のチェックが出るかを見る。
- 否定系テスト（AIが書き換えられないこと等の自動検証）は次の承認（X-05-2）から。
- **GO済み基準は Phase 2-B-5 / `83d35bc` のまま**（CI はアプリの動きを変えないため）。
