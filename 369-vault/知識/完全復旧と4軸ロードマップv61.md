# 完全復旧と4軸ロードマップ v6.1 — 「消えた」の正体は配信、そして競合を証拠で測る回

2026-07-11。利用者が「4つの機能がサイドバーから消えた」と報告した回。調べると**コードは消えていなかった**。正本は GitHub docs（audit176・roadmap80・PR #14）。

## 何が起きていたか

- 利用者の Vercel 画面のメニュー（63項目）は **main の内容そのもの**だった。欠落した4機能（Growth管制塔・広告分析・SEO・3Dオフィス）は、統合ブランチ（PR #12）に**コードとして存在**し、Preview という別の配信面にだけ出ていた。
- つまり「実装が消えた」のではなく「**新しい配信がまだ本番相当画面に出ていない**」という配信系譜（deployment lineage）の問題。GitHub の Deployments API で Production=main・Preview=統合 と固定して確定した。

## v6.1 でやったこと

1. **専用の復旧ブランチ**（`claude/full-recovery-v61`）を統合 head から作り、そこで直した（main は人間 GO まで触らない）。
2. **秘密マスクの再監査**: Codex が「まだ漏れる」と指摘した3経路（エスケープ引用符付きの値・改行直後に続く Cookie 値・裸の session）を、**まず失敗するテストで再現**してから直した。17個の敵対的入力すべてが隠れることを確認。
3. **クラッシュ残骸の誤判定を修正**: 3時間前に落ちた実行が、新しい実行を「二重だ」と永久に潰すバグを直した（残骸は競合に数えない）。
4. **AI社員と3Dオフィスの人物を1つの正本に統一**: 人物・見た目・プロフィールは `getAiCharacter` だけ、状態は3Dと同じ計算だけ。一覧・詳細・3Dが互いにリンクでつながり、同じ社員を指す。
5. **メニューの静的契約テスト**: 67項目・重複ゼロ・全項目に実ページがある・4機能を消すとテストが赤くなる。
6. **ビルド識別バッジ**: OWNER/ADMIN が「今どの配信（本番/プレビュー）・どの版か」を一目で分かるように（秘密は出さない）。

## 思想メモ（[[369の思想と世界観]] との接続）

- **「消えた」を鵜呑みにしない**: 症状（メニューに無い）と原因（配信系譜）を分ける。コード比較で削除0を確認してから結論した。
- **「直した」は赤テストから**: 指摘された穴は、まず再現して赤くしてから緑にする。宣言ではなく証拠。
- **競合を名前で比較しない**: 「セールスフォースと同じ」「マネーフォワードを超えた」とは言わない。capability ごとに方式（自前/AI補助/外部接続/未計画/人間専任）と証拠段階を付ける。会計の実送金・実課金・税務断定はしない——検知・予測・下書き・可視化・承認要求まで。
