# 既知の落とし穴とローカル検証

> 目次に戻る → [[index]] ／ 関連 → [[アーキテクチャ概要]] ・ [[安全第一の哲学]]

## 踏み抜きやすい罠

- **autoprefixer を PostCSS に戻さない**: `tailwindcss` の jiti 設定ローダ（peer に tsx）が `browserslist` のロードを壊し `next build` が失敗する。`postcss.config.cjs` は tailwindcss のみ。tailwind/postcss 設定は **`.cjs`（CommonJS）**で維持（web は `"type":"module"`）。
- **パッケージ間importに `.js` 拡張子を付けない**（付けると webpack が解決失敗）。
- **Next 15 では `params`/`searchParams`/`cookies()` が async**。`await` する。
- **`prisma migrate reset`（`pnpm db:reset`）は Claude Code 実行を検知して安全ガードで拒否**される。クリーン再投入は `pnpm db:seed`（内部で TRUNCATE ... CASCADE 後に再生成）を使う。
- **lint/typecheck に migrate/seed を足さない**（前段は `prisma generate` のみ）。

## 環境制約（BLOCKERS）

- **B-01 Prisma エンジンの自動DL不可（ECONNRESET）**: サンドボックスのネットワークでエンジン取得が失敗する。回避策＝ローカルに既存のエンジンを指す環境変数を設定してから検証を走らせる：
  ```
  ENGINES_DIR="node_modules/.pnpm/@prisma+engines@<ver>/node_modules/@prisma/engines"
  export PRISMA_QUERY_ENGINE_LIBRARY="$(pwd)/$ENGINES_DIR/libquery_engine-debian-openssl-3.0.x.so.node"
  export PRISMA_SCHEMA_ENGINE_BINARY="$(pwd)/$ENGINES_DIR/schema-engine-debian-openssl-3.0.x"
  export PRISMA_CLI_QUERY_ENGINE_TYPE=library
  export PRISMA_ENGINES_MIRROR="file://$(pwd)/$ENGINES_DIR"
  export PRISMA_SKIP_POSTINSTALL_GENERATE=1
  export SKIP_DB_SETUP=1
  ```
  ※ Vercel 本番ビルドは `rhel-openssl-3.0.x` 版を同梱する別系統なので無関係。
- **B-02 統合テスト用 Postgres は手動起動が必要**（ローカルでネイティブ起動）。単体 `pnpm test` には含めない。
- **Playwright のブラウザDL**はサンドボックスのネットワークで失敗しがち。代わりに署名Cookieを使った HTTP スモークで主要画面の 200 を確認する運用がある。
- ローカル検証環境では Postgres/Redis をネイティブ起動（例 `/var/lib/pg-ikezaki`、`redis-server --daemonize`）。

## 検証の考え方

- 変更したら `pnpm test && pnpm typecheck && pnpm lint`（＋必要なら build）を通す。
- **検証していないものを成功扱いしない。** docs-only（コード非変更）なら「docs-only・コード非変更のため未実行」と明記する。
- ワンショット検証: `./scripts/verify.sh`（db:generate/typecheck/lint/unit/build）。B-01 の env を先に export しておく。

## コンテナは揮発する

- リモート実行環境はエフェメラル。**未pushコミット・ローカル生成物は失われうる**。残したいものは commit & push する。
