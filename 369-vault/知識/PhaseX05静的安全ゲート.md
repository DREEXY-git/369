# Phase X-05-3 — 「存在してはいけないもの」を機械が見張る静的安全ゲート

> 出典: `369` リポジトリ `docs/audit/68_phase_x05_3_company_brain_static_safety_gate.md`（判定 GO・commit-only）。

## 何をしたか

- これまで毎回の作業後に**手動のチェックリスト**で確認してきた「できてはいけないことが、コード上に存在しないこと」を、**検査プログラム1本に自動化**し、CI（自動チェック）に組み込んだ。
- 見張る対象（会社の頭脳3テーブルの安全境界）:
  1. **物理削除が存在しない**こと（消せるのはアーカイブだけ）
  2. **機密ラベルが通常/社内限の2択のまま**であること（高機密の解禁は別承認）
  3. **外部AI送信を許可するボタンが画面のどこにも存在しない**こと（147ファイルを走査）
  4. **「AIは書き換えられない」共通判定が正しく使われ続けている**こと（こっそりコピー定義が復活していないこと）
- 検査が失敗すると、**どの約束が破れたかを日本語で表示して赤信号**になる。

## この変更の意味

- 設計（[[PhaseX05CI否定系テスト設計]]）の8つの守り対象のうち、**★1（AIロール拒否=テスト）＋★3・★4・★5（静的ゲート）の4つが自動化完了**。
- このプロジェクト独特の「ゲート検査文化」（毎回手動で grep 確認してきた規律）が、そのまま**仕組み**になった。
- アプリの動きは何も変えていない（アプリ本体のコードに差分ゼロ・追加パッケージもゼロ）。
