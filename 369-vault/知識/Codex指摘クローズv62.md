# Codex 指摘クローズ v6.2 — レビューで返ってきた「まだ漏れる」を設計から直す回

2026-07-12。v6.1 の復旧を Codex が独立レビュー（Track B）して **CHANGES REQUIRED**（要修正）が返ってきた回。正本は GitHub docs（audit176 v6.2 訂正・roadmap80 §3/§9・PR #14）。

## 何が返ってきたか

Codex が「直したはずの秘密マスクがまだ漏れる」ことを実コードで再現した:
`{"password":"abc\"SECRET_TOKEN"}` を通すと `{"password":[masked]SECRET_TOKEN"}` になり **SECRET_TOKEN が残った**（High）。ほかに、クラッシュ判定の規則が作成前と作成後で食い違う・ビルド表示バッジの権限が広すぎる・AI社員の一致テストが8名中1名だけ、という指摘（P2）も。

## v6.2 でどう直したか

1. **秘密マスクを設計から作り直した**: これまでは「エスケープされた引用符を全部ふつうの引用符に戻す」という乱暴な前処理をしていて、それが値の中の引用符まで壊し、秘密の後半を残していた。v6.2 は**引用符とエスケープの状態を1文字ずつ理解して読む「スキャナ」**に置き換え。受入例・入れ子・配列・接尾辞・巨大入力すべてで「秘密が一文字も残らない」ことをテストで直接確かめる（普通の文章は消しすぎない）。
2. **クラッシュ判定を1つの正本に統一**: 「実行中か・残骸か」を判定する関数を1つにし、作成前チェックと作成後チェックが**まったく同じ規則**を使うようにした（RUNNING も QUEUED も、開始時刻不明も、安全側に揃える）。
3. **ビルド表示を社長/管理者だけに**: 権限（admin:read）だと役員や閲覧専用にも見えてしまうので、**役割そのもの（OWNER/ADMIN）**で限定し、他の役割にはサーバーから情報自体を送らない。
4. **AI社員8名全員の一致を証明**: 一覧→詳細→3Dオフィスの往復を8名で確認。存在しない架空IDではなく、**本物の別会社（別テナント）のレコード**を作って404になることを確かめ、後片付けもする。
5. **メニュー契約を強く**: 67項目の並びを明示リストで固定し、1つを別ページに差し替えても件数が同じでも失敗するように。
6. **証拠の言い過ぎを是正**: 「schema があるだけ」を「本番実証済み」と書かない語彙に整理。競合機能は Function 単位で「未検証」に格下げ。

## 思想メモ（[[安全第一の哲学]] との接続）

- **レビューは減点ではなく設計のやり直しの合図**: 「継ぎ足しの正規表現」が破綻したら、正規表現を足すのではなく構造処理（スキャナ）に替える。
- **[masked] があるかではなく、秘密が残らないかを見る**: テストは「隠した印」ではなく「元の秘密文字列が一切現れないこと」を確かめる。
- **競合は名前で比べない**: 「セールスフォースと同じ」「マネーフォワードを超えた」とは言わず、機能ごとに方式（自前/AI補助/外部接続/後回し/人間専任）と証拠段階で表す。会計の実送金・税務断定・採否決定は人間専任のまま。

## 全体の進め方（人間方針）

- **まず安全な MVP をリリース → その後、段階的に全機能を完成**。最初の拡張は各カテゴリの AI 活用 DX と Marketing（Wave 1）。以降 Wave 2=Salesforce系、Wave 3=会計、Wave 4=人事労務、Wave 5=外部連携。将来対象を「やらない」で捨てず「後回し（DEFERRED）」にする。
