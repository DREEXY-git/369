# 残存欠陥クローズと統合 v5.9 — 「直した」を再検証で証明する回

2026-07-11。v5.8 で Codex に指摘された「完了と言ったのに残っていた欠陥」を、継ぎ足しではなく設計から直し、4ストリームを1本の統合ブランチに束ねた回。正本は GitHub docs（roadmap69§0追補2・PR #10/#12 の Conversation・各ストリームの修正 commit）。

## 何が起きたか

1. **Codex PR #10 を受領**（v5.8 独立再レビュー + 台帳検証）: Codex 環境で原典（期待 raw SHA-256 一致）による
   生成器 `--check` が成功していた記録を確認し、状態を **SOURCE_RECHECK_VERIFIED_BY_CODEX** へ更新。
   ただし Phase 3 の正式完了・main 統合・人間 GO とは分離（引き続き HOLD）。検証主体が Codex であることを明記し、
   「Claude が独立再生成済み」とは書かない。
2. **High-1（秘密マスク漏れ）を正規化先行設計で全面修復**: 旧実装は quoted JSON キー（`"authorization":`）や
   折返しヘッダ（`Cookie:\n sid=...`）を取り逃していた。v5.9 は「①CRLF/折返しの正規化 → ②URL→quoted キー→
   ヘッダ行→key=value→スキーム→JWT→鍵生値の順で redact → ③1行化＋上限」という段階設計へ書き換え、
   Codex 再現3経路を含む否定テスト8件で「秘密が残らない」ことを証明した。
3. **High-2（BullMQ 統合証拠）は EVIDENCE_GAP を維持**: CI 基盤に Redis が無く、unit green だけで
   「完全解消」とは記録しない。証拠が足りないものは足りないと書く。
4. **成果台帳の数え方を正した**: M7 = 同じ Run の承認と Gate を二重に数えない（判断単位の集合和）。
   M8 = 件数・時間・円という単位の違うものを合算しない（区分×単位の2段バケット）。どちらも否定テスト付き。
5. **3Dオフィスの視覚欠陥**: M11 = ネームプレートをクリックしても選択できなかった → 当たり判定へ追加し、
   読み取り専用フック＋実クリックの回帰 e2e で証明（CI 実機で 104 passed）。M9 = モバイルのプロフィール証拠画像に
   ヘッダーが重なる → 撮影中のみヘッダーを外し、全項目が目視できる artifact にした。
6. **claude/integration-v59**: feature → A → B → C → D を non-force で統合。競合2ファイル（nav / barrel export）は
   「全導線・全 export を残す」方針で解消し、PR #12（Draft・feature 向け・main 非対象）として提出。

## 思想メモ（[[369の思想と世界観]] との接続）

- **「直した」は宣言ではなく再検証**: v5.8 の教訓（完了と言った後に Codex が欠陥を再現）を受けて、
  v5.9 は否定ケースを先に列挙し、それが全部落ちることを確認してから完了と書いた。
- **数えられないものを数えない**: 削減時間や成果は、単位が違えば足さない・同じ判断は二度数えない。
  経営の数字は「大きく見せる」より「正しく見せる」。
