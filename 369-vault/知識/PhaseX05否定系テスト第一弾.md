# Phase X-05-2 — 「AIが書き換えられないこと」を自動テストで守り始めた

> 出典: `369` リポジトリ `docs/audit/66_phase_x05_2_negative_tests_ai_role.md`（判定 GO・commit-only）。
> 関連: [[PhaseX05CI否定系テスト設計]] / [[PhaseX05CI実走確認]] / [[Phase2A3b1安全補正]] / [[Phase2B完了]]

## 何をしたか

- **「AIは会社の頭脳（会社方針・商品カタログ・営業プレイブック）を書き換えられない」という一番大事な約束を、自動テストで守り始めた。**
- 設計（[[PhaseX05CI否定系テスト設計]]）で特定した最大の穴: この約束を守る判定コードは3つのファイルに手書きコピーされているだけで、テストが1本も無かった。
- 今回: 判定を共通部品1つにまとめ（コピー3つを廃止）、**否定系テスト5本**を追加した。
  - AIロール（AI社員・AI補助）は人間扱いされない
  - 人間とAIが混ざった場合も安全側で拒否
  - 権限が空でも拒否
  - **「権限表の上ではAIも下書き作成権限を持っている」という前提そのものもテストで固定**（将来誰かが権限表を変えたらテストが気づく）

## 動きは変わっていないことの証明

- テスト216本 green・型チェック・lint・ビルド green。
- さらに**ブラウザ実操作テスト18本（作成フロー含む）も全部 green** — 画面の動きが1つも変わっていないことを実測で確認した。

## この変更の意味

- CI（[[PhaseX05CI実走確認]]）が動いているので、**今後は GitHub にコードが上がるたびに「AIが書き換えられないこと」が自動で検証され続ける**。
- 品質の柱が「人の注意」から「仕組み」へ、もう一段移った。
