## 結論（非エンジニア向け）

- 369 OS の現在の強みは **CRM/商談/見積/請求の縦切り＋AI ガバナンス（承認・監査・封印）**。
  Salesforce 級の領域では「取引先360・商談・リード・見積・請求」が部分実装、
  「売上予測・自由レポート・外部メール連携・拡張マーケット」は未実装/外部連携候補。
- **会計（MoneyForward/freee 級）は schema 承認待ちが本丸**（仕訳候補 read-only が最初の1本）。
  銀行明細の取込は自前でなく公式 API 連携（人間 Gate）が本筋。
- **人事・労務と税務は「AI が断定しない」を恒久ルール**として、台帳・境界テストから進める。
  採否・評価・支給の確定は永久に人間。
- **電子帳簿保存は「対応済み」と宣言しない**。保管方針の正本化 → schema → 専門家確認（人間 Gate）の順。

## 6区分の意味

実装済み（Draft+CI まで）／部分実装／未実装／ROADMAP_ONLY（設計のみ）／外部連携候補（公式 API・人間 Gate）／人間Gate（恒久的に人間が最終判断）。

## 次の一手（優先順）

1. 商談ステージ遷移の監査＋実測ファネル（schema 不要）
2. C22 の下書き保存縦切り（既存 ContentAsset 再利用）
3. schema 承認後: 仕訳候補 read-only（P5-ACC-01）→ 紹介台帳本体 → 勤怠最小

「競合同等以上」「完成」は宣言しない。分類の根拠は各社の公式公開資料の一次情報のみ。
