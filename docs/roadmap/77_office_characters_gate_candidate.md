## 1. 設計原則

- **アセット制約**: 外部画像・外部フォント・CDN・既存作品（FF 等）の実アセットは使用不可（CI/CSP・ライセンス）。
  → すべて**手書きベクター（SVG ポートレート）＋ Three.js プリミティブ合成**のオリジナルで「上品な JRPG 調」を実現。
  実在人物・既存キャラクターの模倣はしない（全員オリジナル設定）。
- **設定と実測の分離（証拠規律の維持）**: プロフィール（名前・性格・スキルレベル・クセ・ミス・評価コメント）は
  **キャラクター設定（ペルソナ）**であり、稼働状態・実行回数などの**実測（deriveAgentState/read model 由来）**とは
  別セクションで表示し、「設定」であることを UI に明記。評価コメントにも（設定）を付す。
  → v5.6 の Outcome 証拠5区分・「捏造しない」原則と矛盾しない。
- **schema 変更なし**: プロフィールは `packages/shared/src/ai-characters.ts` の純データ（AIAgent.key で対応・
  未登録 key は「設定未作成」フォールバック）。DB・seed・RBAC・labels 不変。
- **read-only 不変**: 実行・承認・削除・外部送信の導線は追加しない。
