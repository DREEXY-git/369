- 経緯: 初回「全てGO」→ 再確認「表示されない」（GO記録を main 反映前に差し止め・HOLD訂正）→ 詳細実測で症状をナビ表示のみに特定 → read-only 原因調査で repo 側の潔白確認（nav.ts に行あり・フィルタなし）→ **本再実測でハードリロードにより解消**。

- 教訓（vault にも記録）: ①HOLD→差し止め→調査→再実測GO の流れが3例目として機能 ②今後の本番確認手順に「NG時はまずハードリロード」を組み込み済み（doc49 の運用）③smoke がナビ表示を未検証だった盲点は改善候補として残る（ナビ表示検証の追加は別承認・2-B-4 以降で検討）。
