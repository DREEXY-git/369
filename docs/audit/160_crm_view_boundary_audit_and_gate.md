# 160. CRM 閲覧境界 全経路監査＋実装前 Gate — docs/roadmap/61 の記録

## 1. 非エンジニア向け要約

- 顧客情報（CRM）の「見てよい人だけが見られる」境界を全経路で調べました。**7つの穴**が実測で確定しました:
  1. 顧客一覧が、権限チェックなしで**全顧客（機密ラベル付き含む）を全員に表示**
  2. 顧客詳細が、データを**全部取得してから**閲覧可否を判定（順序が逆）
  3. 閲覧拒否の画面に**顧客名が表示**される
  4. 編集画面は閲覧チェックなしで顧客情報を表示
  5. **タイムライン／インサイト画面には閲覧可否判定が無く**、詳細で拒否された人も直URLで全履歴を読める
  6. インサイト画面は、保存済みが無いと**ページを開いただけで顧客名＋履歴を AI に送る**構造（現在は疑似AIなので実害なしだが、本物のAIキーを設定した瞬間に外部送信になる）
  7. 編集アクションに機密ラベル判定が無く、**見られないはずの顧客を編集できる**
- Contact（担当者連絡先）は、データの器はあるが**画面・処理が一切存在しない**（`NO_DIRECT_RUNTIME_SURFACE`）ことを証明しました。
- 実装前チェック（Gate）は全項目合格: **既存の権限部品（hasPermission・canAccessLabel・AccessDenied・ABAC）だけで、DB 設計・権限定義・デモデータを一切変えずに全部塞げます**。
- doc125 の HOLD はこの実装完了後に追記クローズします（「CRM 全体完成」とは呼びません）。

## 2. 判定

**Gate 全 PASS・STOP 非該当**。変更予定はコード7ファイル＋e2e 1 spec のみ。label 拒否系の負系 e2e は seed 制約（全顧客 NORMAL・権限なしログイン不在・seed 変更禁止）により作成不能のため、純エンジン unit＋監査表＋敵対的レビューで代替し、未実施理由を roadmap61 §4 に記録済み。
