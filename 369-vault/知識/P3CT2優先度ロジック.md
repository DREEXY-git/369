## これは何か

- Control Tower（成長機会の「見るだけ」画面）で、**どのカードを上に出すかの並び順ロジックを賢くした**作業です。画面の見た目はほぼ変えていません。
- 変えたのは計算用の純粋なロジック（DBに触れない部品化されたコード）だけ。今までは「重要度＋件数」を足すだけでしたが、**「重要度 × (基礎＋緊急度) × 信頼度」**という説明できる式に精緻化しました。

## どう賢くなったか（やさしい言い換え）

- **重要度**: その機会が経営にどれだけ効くか。
- **緊急度**: 件数が多いほど上がるが、一定件数で頭打ち（過剰に釣り上げない）。
- **信頼度**: そのデータの確からしさ（会社の頭脳の補充候補や件数だけの指標は低め）。
- 各カードに「なぜこのスコアか」の内訳（scoreBreakdown＝重要度・緊急度・信頼度）を持たせ、説明できるようにしました。スコアは上限100・件数0なら0・同点は重要度で安定した並びになります。
- 乱数や現在時刻に依存しない（同じ入力なら必ず同じ結果）作りです。

## 変わらない約束

- **担当者（財務権限なし）に金額（原価・粗利・未回収）の実値を見せない方針は不変**。金額系カードはデータ構造レベルで件数を持たず、「原価・粗利は財務閲覧権限が必要です（機密情報）。」と表示されます。
- 顧客の個人情報も増やしていません（件数だけ）。
- DB 設計（schema/migration）・権限（RBAC）・デモデータ（seed）は一切変更なし。外部送信・実LLM・AIコスト・課金・本番反映もなし。
- 検証: 単体テストは新規7件を足して 278 件すべて合格。型チェック・lint・安全チェックも緑。画面のE2Eは push 後の CI で確認（74件・0失敗の見込み）。
- 前提となる CI は P3-CT-1 push 後の run 28944487139（74 passed / 0 failed・Control Tower の2件も緑）。
