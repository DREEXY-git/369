# 369の思想と世界観

> 目次に戻る → [[index]] ／ 関連 → [[安全第一の哲学]] ・ [[AIの役割と境界]]

## 一言で

**中小企業の経営を、AIと一緒に「安全に」自走させる統合OS。**
その中核に、地図×AIで新規開拓を回す **LeadMap AI**（リード抽出 → AI分析 → 個別営業メール → 承認 → 送信 → 追客 → 商談化）を置く。

## 何を作っているのか

- **IKEZAKI OS**: 中小企業向けの統合AI経営OS。経営／営業／CRM／会計／財務／人事／在庫／会議／ナレッジ／AI社員を1つに統合する。
- 目指す姿は **Self-Evolving Agentic Business OS** — 補助型AI・自立型AI・管理職AI・経営AIが業務を支援・代行・報告・改善していく経営基盤。
- ただし現在地はあくまで **実用MVP**。巨大な未完成より、「動く薄い縦切り」を積む。

## 世界観・ビジョン

- 中小企業には、大企業のような専任の経営企画・情報システム・法務がいない。そこを **AIが埋める**。
- AIは「魔法の一撃」ではなく、**日々の業務に溶け込み、記録し、下書きし、報告し、少しずつ改善する**もの。
- APIキーが無くても **FakeLLM / DemoMapProvider** でフル機能がデモできる。「触れば分かる」ことを重視し、**空画面を作らない**。

## 核心的な価値観

1. **薄い縦切り優先**: 重要テーブル／API／UI／デモデータ／権限／監査ログを必ず1本通す。CRUD＋権限＋監査＋デモデータの一気通貫を、巨大な未完成より優先する。
2. **安全 > 速度**: 経営・お金・個人情報に触れるからこそ、壊さないことを最優先にする。→ [[安全第一の哲学]]
3. **AIは断定助言しない**: 法務／税務／労務／財務を断定せず、リスク・確認観点・専門家相談候補に留める。→ [[AIの役割と境界]]
4. **証拠を残す**: AI回答には可能な限り 参照元／根拠／信頼度／推奨アクション を残す。「完了しました」だけを信じない。
5. **将来の収益化は前提を整えるだけ**: 利用量（UsageEvent）は貯めるが、**今は課金しない**。→ [[UsageEvent非課金台帳]]

## Monetization（収益化）への態度

- UsageEvent は「利用量可視化 → AIコスト管理 → 従量課金の前提 → プラン設計 → 安全な monetization layer」への布石。
- しかし **現時点では課金・決済・サブスク・請求額計算・usage cap・プラン連動を一切しない**。
- 利用量はあくまで **非課金の記録・監査・可視化**。`billing = usage_only`。金額カラムを持たない。
- 実課金は Phase 1 完了後・別設計・人間承認が前提。急がない。
