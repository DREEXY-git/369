# 状態管理修復プロンプトの型

> 目次に戻る → [[index]] ／ 関連 → [[アーキテクチャ概要]] ・ [[意思決定ログ]]

## 用途

`PROGRESS.md` や `CURRENT_STATE.md` に紛れ込んだ **陳腐化・一時状態・曖昧表現**を、docs だけ最小修正で安定化させる型。「push後・次回作業後にも古くならない」表現に直す。

## 直す対象（アンチパターン）

- **固定した現在値**: `HEAD: <sha>` / `origin/main: <sha>` / `unpushed commits: なし` / `working tree: clean` / `branch: <名>` を本文に固定 → コミット/pushで即古くなる。
- **一時状態の永続化**: `push 未実施` / `人間承認待ち` / `ローカル実装` / `今回` などを永続docsに残す。
- **誤読される見出し**: 「最新完了 Phase」→ Phase全体が完了と誤読されうる。

## 直し方（原則）

1. **現在値は git を正とする**と明記し、固定値を撤去：
   - `git rev-parse --short HEAD` / `git rev-parse --short origin/main` / `git log --oneline origin/main..HEAD` / `git status --short` を参照させる。
2. **固定してよいのは「基準 commit」だけ**：実装 commit と 完了基準（本番確認GO）commit の2つ。「これは現在HEADではない」と注記する。
3. **見出しを具体化**：「最新の本番確認GO済みプロダクト基準」等、意味が動かない語に。
4. **一時状態は最終報告に書き、docsには残さない。**

## 検証（修正後）

```
- 変更ファイルが対象1ファイルのみ
- 固定 git-state（HEAD:` / origin/main:` / branch:` / unpushed commits: / working tree:）が残っていない
- 現在HEAD系の作業SHAが本文に残っていない
- 一時/曖昧 wording（最新完了 Phase / 今回 / push 未実施 / 人間承認待ち）が残っていない
- 基準 commit（実装 / 完了基準）は残っている
- git diff --check clean / secret 実値なし
```

## 状態管理の役割分担（推奨）

- **PROGRESS.md** = 履歴・判断メモ（積み上げ）。
- **CURRENT_STATE.md** = 現在地の1枚サマリー（今の真実だけ）。
- **usage_event_emit_matrix.md** 等 = 一覧の固定。
- いずれも「反映状態は git refs を正とする」を前提にする。
